## What are Business Process Flows?

### The Concept

Business Process Flows (BPFs) are **visual guides** that help users follow a consistent, repeatable process. They appear as a horizontal bar at the top of model-driven app forms, showing the stages of a process.

## Creating Business Process Flows

### Creation Process

1. **Access the maker portal**
2. **Select table(s)** the BPF will apply to
3. **Design stages and steps**
4. **Add branching logic** (optional)
5. **Configure workflows** (optional)
6. **Activate the BPF**

|--------|--------|

| **Abandon** | Mark process as cancelled/stopped |

## Common Exam Scenarios

### Scenario 1: Tracking Stage Duration
**Question:** "Managers need to report on how long opportunities spend in each stage."

**Answer:** Query the **BPF table** which automatically tracks stage changes and timestamps. Create views or Power BI reports on this table.

### Scenario 2: Spanning Multiple Tables
**Question:** "A process starts with Lead qualification and continues through Opportunity to Quote."

**Answer:** Create a BPF that includes Lead, Opportunity, and Quote tables (max 5 tables allowed).

### Scenario 3: Different Paths for Different Deals
**Question:** "Large deals (>$100K) need executive approval, small deals don't."

**Answer:** Use **branching logic** based on the deal value field. Create separate stages for the large deal path.

### Scenario 4: Enforcing Required Data
**Question:** "Users must enter budget information before advancing past the Qualify stage."

**Answer:** Add Budget as a **required step** in the Qualify stage. Users cannot advance until completed.

### Scenario 5: Automating Stage Actions
**Question:** "When a deal enters the Contract stage, automatically create a review task for Legal."

**Answer:** Configure a **workflow on stage enter** for the Contract stage that creates the task.

### Scenario 6: BPF Not Appearing
**Question:** "Users report the BPF isn't showing on the form."

**Answer:** Check:
1. BPF is **activated**
2. User has **read privilege** on the BPF
3. BPF is included in the **model-driven app**
4. Record matches any BPF conditions

### Scenario 7: Reporting on Process Metrics
**Question:** "Need to identify process bottlenecks."

**Answer:** Use the **BPF table** to analyze:
- Average duration per stage
- Records stuck in stages
- Completion rates

## Key Takeaways for the Exam

1. **BPF creates its own table** - for tracking, reporting, automation
2. **Maximum 5 tables** per BPF
3. **Maximum 30 stages** per BPF
4. **Required steps** block stage advancement
5. **Branching** uses conditions based on field values
6. **Stage workflows** run on enter/exit
7. **Action steps** = buttons that trigger workflows
8. **Multiple BPFs** can be active on same record
9. **BPF security** is separate from table security
10. **Power Automate** can trigger on BPF table changes
